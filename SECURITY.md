# 安全政策

## 报告安全漏洞

如果你发现了 StreamerHelper 的安全漏洞，请**不要**公开创建 Issue。

### 私密报告

请发送邮件至项目维护者，包含：

- 漏洞描述
- 影响范围
- 复现步骤
- 建议的修复方案（如果有）

我们会在收到报告后尽快回复，并制定修复计划。

### 支持的版本

我们只会为最新版本提供安全更新。请确保你使用的是最新版本。

## 安全最佳实践

### 部署时

1. **修改默认密码** - 更改所有默认密码和密钥
2. **使用 HTTPS** - 生产环境务必使用 HTTPS
3. **限制访问** - 使用防火墙限制不必要的端口访问
4. **定期更新** - 及时更新依赖包和系统
5. **备份数据** - 定期备份数据库和重要文件

### 配置文件

- 配置文件 `~/.streamer-helper/settings.json` 包含敏感信息
- 确保该文件权限设置为 `600`（仅所有者可读写）
- 不要将配置文件提交到版本控制

### API 密钥

- B站等平台的 API 密钥存储在数据库中
- 建议使用环境变量管理敏感信息
- 定期轮换密钥

## 依赖管理

我们会定期扫描依赖项中的已知漏洞。如果你发现相关问题：

1. 检查是否有可用更新
2. 在 Issues 中报告（非安全问题）
3. 提交 PR 修复
